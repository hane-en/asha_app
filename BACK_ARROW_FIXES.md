# إصلاح مشكلة سهم العودة في صفحة الطلبات

## 🔧 المشكلة:

### **الوصف**:
- عند الدخول لصفحة الطلبات من القائمة الجانبية (Drawer)
- والخروج للصفحة الرئيسية عبر سهم العودة
- تظهر رسالة "الصفحة غير موجودة"

### **السبب**:
- صفحة الطلبات لا تحتوي على سهم العودة في AppBar
- أو أن التنقل غير صحيح

## 🛠️ الحل:

### **إضافة سهم العودة لصفحة الطلبات**:

#### ✅ **صفحة الطلبات** (`lib/screens/user/booking_status_page.dart`)

**قبل التحديث**:
```dart
appBar: AppBar(title: Text('حالة الطلبات')),
```

**بعد التحديث**:
```dart
appBar: AppBar(
  title: const Text('حالة الطلبات'),
  leading: IconButton(
    icon: const Icon(Icons.arrow_back),
    onPressed: () {
      Navigator.pushReplacementNamed(context, RouteNames.userHome);
    },
  ),
),
```

## 📱 الصفحات المحدثة:

### ✅ **1. صفحة الطلبات** (`lib/screens/user/booking_status_page.dart`)
- **التحديثات**:
  - إضافة سهم العودة في AppBar
  - استخدام `Navigator.pushReplacementNamed` للتنقل للصفحة الرئيسية
  - التأكد من استخدام `RouteNames.userHome` الصحيح

### ✅ **2. صفحة المفضلة** (`lib/screens/user/favorites_page.dart`)
- **الحالة**: ✅ تحتوي على سهم العودة
- **التنقل**: صحيح للصفحة الرئيسية

### ✅ **3. صفحة البحث** (`lib/screens/user/search_page.dart`)
- **الحالة**: ✅ تحتوي على سهم العودة
- **التنقل**: صحيح للصفحة الرئيسية

### ✅ **4. صفحة الإشعارات** (`lib/screens/user/notifications_page.dart`)
- **الحالة**: ✅ تحتوي على سهم العودة
- **التنقل**: صحيح للصفحة الرئيسية

## 🔄 التنقل المحدث:

### **سهم العودة في جميع الصفحات**:
```dart
leading: IconButton(
  icon: const Icon(Icons.arrow_back),
  onPressed: () {
    Navigator.pushReplacementNamed(context, RouteNames.userHome);
  },
),
```

### **المزايا**:
- ✅ تنقل صحيح للصفحة الرئيسية
- ✅ لا تظهر رسالة "الصفحة غير موجودة"
- ✅ تجربة مستخدم متسقة
- ✅ استخدام `pushReplacementNamed` لتجنب تراكم الصفحات

## 🎯 النتيجة النهائية:
