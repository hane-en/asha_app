# شرح هيكل مجلد lib ووظيفة كل ملف

## main.dart
نقطة البداية للتطبيق، يحتوي على تهيئة التطبيق وتهيئة الثيمات ومسارات البداية.

## config/
- **config.dart**: إعدادات التطبيق العامة (روابط API، إعدادات الألوان، الرسائل الافتراضية...)

## models/
- **ad_model.dart**: نموذج بيانات الإعلان.
- **booking_model.dart**: نموذج بيانات الحجز.
- **service_model.dart**: نموذج بيانات الخدمة.
- **user_model.dart**: نموذج بيانات المستخدم.

## routes/
- **app_routes.dart**: تعريف جميع مسارات التنقل بين الصفحات.
- **route_names.dart**: تعريف أسماء المسارات كـ constants.
- **route_arguments.dart**: تعريف أنواع البيانات التي تُمرر بين الصفحات.

## screens/
يحتوي على جميع صفحات التطبيق، مقسمة حسب نوع المستخدم أو الوظيفة:

### admin/ (صفحات الإدارة)
- **admin_dashboard.dart**: لوحة تحكم المسؤول، تعرض إحصائيات عامة عن النظام مثل عدد المستخدمين، الحجوزات، الخدمات، والأرباح.
- **admin_home_page.dart**: الصفحة الرئيسية للمسؤول بعد تسجيل الدخول، نقطة انطلاق لإدارة النظام.
- **admin_settings_page.dart**: إعدادات المسؤول، لتغيير بيانات المسؤول أو إعدادات النظام العامة.
- **all_bookings_page.dart**: عرض جميع الحجوزات في النظام مع إمكانية البحث والتصفية.
- **delete_user_page.dart**: واجهة لحذف مستخدم من النظام بشكل نهائي.
- **edit_user_profile.dart**: تعديل بيانات مستخدم (اسم، بريد، رقم... إلخ) من قبل المسؤول.
- **join_requests_page.dart**: إدارة طلبات انضمام مزودي الخدمة الجدد (قبول/رفض).
- **manage_ads.dart**: إدارة جميع الإعلانات المنشورة في النظام (تعديل/حذف/إخفاء).
- **manage_bookings.dart**: إدارة الحجوزات (تأكيد، إلغاء، مراجعة تفاصيل الحجز).
- **manage_comments.dart**: مراجعة التعليقات والتقييمات على الخدمات مع إمكانية الحذف أو الرد.
- **manage_services.dart**: إدارة جميع الخدمات المقدمة في النظام (إضافة/تعديل/حذف).
- **manage_users.dart**: إدارة المستخدمين (عرض، تعديل، حذف، حظر... إلخ).
- **search_user_page.dart**: البحث عن مستخدمين عبر الاسم أو البريد أو رقم الهاتف.
- **user_details_page.dart**: عرض تفاصيل مستخدم معين (بياناته، حجوزاته، نشاطه).
- **user_list_page.dart**: قائمة بجميع المستخدمين مع خيارات الإدارة.

### auth/ (صفحات المصادقة والتوثيق)
- **forgot_password_page.dart**: استرجاع كلمة المرور عبر البريد أو الهاتف.
- **login_admin.dart**: تسجيل دخول المسؤول (admin) فقط.
- **login_page.dart**: تسجيل دخول المستخدم العادي (user).
- **provider_login.dart**: تسجيل دخول مزود الخدمة (provider).
- **provider_register.dart**: تسجيل مزود خدمة جديد مع بياناته ونوع الخدمة.
- **reset_password_page.dart**: إعادة تعيين كلمة المرور بعد التحقق.
- **send_verification_code_page.dart**: إرسال رمز تحقق للهاتف (SMS) أثناء التسجيل أو استرجاع الحساب.
- **signup_page.dart**: إنشاء حساب جديد (user أو provider) مع اختيار نوع الحساب.
- **verify_page.dart**: إدخال رمز تحقق (عادةً 4 خانات) للتحقق من الهوية.
- **verify_sms_code_page.dart**: إدخال رمز تحقق SMS (عادةً 6 خانات) للتحقق من رقم الهاتف.

### provider/ (صفحات مزود الخدمة)
- **add_ad_page.dart**: إضافة إعلان جديد عن خدمة أو عرض خاص.
- **add_service_page.dart**: إضافة خدمة جديدة يقدمها المزود.
- **booking_details_page.dart**: عرض تفاصيل حجز معين خاص بالمزود.
- **edit_ad_page.dart**: تعديل إعلان سابق.
- **edit_service_page.dart**: تعديل بيانات خدمة موجودة.
- **manage_my_ads.dart**: إدارة جميع إعلانات المزود (تعديل/حذف/إخفاء).
- **manage_my_services.dart**: إدارة جميع خدمات المزود (تعديل/حذف).
- **my_ads_page.dart**: قائمة إعلانات المزود الحالية.
- **my_bookings_page.dart**: قائمة الحجوزات التي تخص المزود.
- **my_services_page.dart**: قائمة الخدمات التي يقدمها المزود.
- **notifications_page.dart**: إشعارات المزود (حجوزات جديدة، تعليقات، تنبيهات... إلخ).
- **provider_analytics_page.dart**: إحصائيات وتحليلات حول أداء المزود (عدد الحجوزات، الإيرادات... إلخ).
- **provider_bookings.dart**: إدارة الحجوزات الخاصة بالمزود (تأكيد، إلغاء، مراجعة).
- **provider_help_page.dart**: صفحة المساعدة والدعم الفني لمزود الخدمة.
- **provider_home_page.dart**: الصفحة الرئيسية لمزود الخدمة بعد تسجيل الدخول.
- **provider_home.dart**: واجهة رئيسية مختصرة للمزود (قد تكون Dashboard مصغرة).
- **provider_pending_page.dart**: صفحة انتظار موافقة الإدارة على حساب المزود.
- **send_massage_page.dart**: إرسال رسالة (غالباً دعم أو تواصل مع العملاء).
- **settings_page.dart**: إعدادات حساب المزود (تغيير بيانات، كلمة مرور... إلخ).

### services/ (صفحات الخدمات العامة)
- **admin_api.dart**: نقطة تواصل مع API الإدارة (غالباً للاستخدام الداخلي).
- **api_service.dart**: نقطة تواصل مع API الخدمات العامة (جلب بيانات، إرسال بيانات... إلخ).
- **auth_service.dart**: نقطة تواصل مع API المصادقة (تسجيل، تحقق، استرجاع كلمة مرور... إلخ).

### user/ (صفحات المستخدم العادي)
- **booking_page.dart**: حجز خدمة جديدة من خلال التطبيق.
- **booking_status_page.dart**: عرض حالة الحجز (مؤكد، ملغي، قيد الانتظار... إلخ).
- **change_password_page.dart**: تغيير كلمة المرور للمستخدم.
- **comment_page.dart**: إضافة أو عرض تعليق/مراجعة على خدمة.
- **delete_account_page.dart**: حذف حساب المستخدم نهائياً.
- **details_page.dart**: تفاصيل خدمة أو إعلان أو حجز.
- **favorites_page.dart**: قائمة الخدمات أو الإعلانات المفضلة للمستخدم.
- **home_page.dart**: الصفحة الرئيسية للمستخدم (عروض، خدمات، إعلانات... إلخ).
- **join_provider_page.dart**: طلب الانضمام كمزود خدمة (تحويل حساب user إلى provider).
- **notifications_page.dart**: إشعارات المستخدم (حجوزات، عروض، رسائل... إلخ).
- **search_page.dart**: البحث عن خدمات أو مزودين أو إعلانات.
- **service_details_page.dart**: تفاصيل خدمة معينة مع إمكانية الحجز أو التقييم.
- **service_list_page.dart**: قائمة بجميع الخدمات المتاحة.
- **settings_page.dart**: إعدادات المستخدم (تغيير بيانات، كلمة مرور... إلخ).
- **user_help_page.dart**: صفحة المساعدة والدعم الفني للمستخدم.
- **user_home_page.dart**: الصفحة الرئيسية للمستخدم بعد تسجيل الدخول (Dashboard).

## services/
- **auth_service.dart**: التعامل مع عمليات المصادقة (تسجيل، دخول، تحقق...)
- **api_service.dart**: عمليات التواصل العامة مع الـ backend (جلب بيانات، إرسال بيانات...)
- **admin_service.dart**: خدمات خاصة بالإدارة.
- **provider_api.dart**: خدمات خاصة بمزود الخدمة.
- **provider_service.dart**: إدارة خدمات مزود الخدمة.

## utils/
- **constants.dart**: ثوابت عامة للتطبيق (ألوان، نصوص ثابتة...)
- **helpers.dart**: دوال مساعدة متنوعة (تنسيق نصوص، تواريخ، أرقام...)
- **validators.dart**: دوال التحقق من صحة المدخلات.

## widgets/
- **ad_widget.dart**: ويدجت عرض إعلان.
- **service_card.dart**: ويدجت عرض خدمة.
- **sidebar_widget.dart**: ويدجت القائمة الجانبية.

---

كل مجلد فرعي داخل screens (مثل user, provider, admin, auth) يحتوي على صفحات متخصصة حسب نوع المستخدم أو الوظيفة، وكل صفحة عبارة عن واجهة مستقلة (شاشة) في التطبيق.

هذا الملف يساعد أي مطور جديد على فهم بنية المشروع بسرعة.
